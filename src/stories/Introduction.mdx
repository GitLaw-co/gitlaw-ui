# GitLaw UI Component Library

Welcome to the **GitLaw UI** component library, built with React and Tailwind CSS. This Storybook showcases the design system extracted from the GitLaw Figma library.

## Design Tokens

The design system uses the following core tokens:

### Semantic Colors

| Token | Value | Usage |
|-------|-------|-------|
| `primary` | `#5E49D6` | Primary brand color, buttons, links |
| `primary-hover` | `#6F5EEC` | Primary hover state |
| `secondary` | `#EBE7FF` | Secondary backgrounds, muted elements |
| `secondary-hover` | `#E2DDFF` | Secondary hover state |
| `destructive` | `#E11D48` | Error states, delete actions |
| `alert` | `#F97316` | Warning states |
| `positive` | `#15803D` | Success states |
| `border` | `#E2DDFF` | Default border color |

### Text Colors

| Token | Value | Usage |
|-------|-------|-------|
| `text-primary` | `#1B1B1F` | Primary text (obsidian/950) |
| `text-secondary` | `#989898` | Secondary/muted text (grey/50) |
| `text-negative` | `#F7F6FF` | Text on dark backgrounds (purple/50) |
| `text-button` | `#5E49D6` | Button text (secondary variant) |

### Icon Colors

| Token | Value | Usage |
|-------|-------|-------|
| `icon-primary` | `#5E49D6` | Primary icon color |
| `icon-secondary` | `#989898` | Secondary icon color |
| `icon-disabled` | `#CFC8F3` | Disabled icon color |
| `icon-negative` | `#F7F6FF` | Icon on dark backgrounds |
| `icon-black` | `#1B1B1F` | Black icon color |

### Typography

The design system uses **DM Sans** as the primary font family with the following sizes:

- `text-xxs`: 10px
- `text-xs`: 12px
- `text-sm`: 14px
- `text-base`: 16px
- `text-lg`: 18px
- `text-xl`: 20px
- `text-2xl`: 24px

### Spacing

| Token | Value |
|-------|-------|
| `xs` / `s` | 4px |
| `m` | 8px |
| `l` | 12px |
| `xl` | 16px |
| `2xl` | 24px |

## Components

### Button
A versatile button component with multiple variants (primary, secondary, outline, ghost, destructive, disabled) and sizes (xs, s, m, l, xl).

### Input
Text input component with support for labels, icons, and multiple sizes.

### Avatar
User avatar component supporting images, initials, icons, and empty states in multiple sizes.

### Badge
Notification badge component with color variants and optional count display.

### Tag
Label/tag component for categorization with icon and close button support.

### Tab
Navigation tab component with badge support and selection states.

### Switch
Toggle switch component for boolean settings.

### ListItem
List item component with multiple left element types (icon, avatar, clause, select, mini-users), secondary text, and selection states.

### Select
Dropdown select component with search functionality, multiple sizes, validation states, and label positioning options.

### TextField
Multi-line text area component with label support, multiple sizes, and validation states.
